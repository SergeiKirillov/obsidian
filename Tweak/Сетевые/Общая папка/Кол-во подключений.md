# Отключение 
## Минусы
- 👉 Требует **администратора обязательно**
## Реализация
для отключения всех пользователей от шары нужны права администратора
```bat
@echo off
net session /delete /y
```
👉 Требует **администратора обязательно**

```PowerShell
Get-SmbSession | Close-SmbSession -Force
```
👉 Требует запуск PowerShell **от имени администратора**

Если цель — не дать пользователям постоянно держать файлы открытыми, лучше настроить автоматическое завершение неактивных SMB-сеансов:

```
net config server /autodisconnect:10
```

# Ограничение на ко-во подключений - при подключении получит ошибку.
## минусы
Пользователь вне ограничений **не сможет подключиться вообще**, и получит отказ на уровне SMB.
Обычно сообщение выглядит так:
- **“No more connections can be made to this remote computer at this time because there are already as many connections as the computer can accept.”**
- или в проводнике Windows:
    - “Нет доступа”
    - “Сетевое имя больше не доступно”


## 1. Ограничение через саму шару (самый простой способ)
Можно задать лимит прямо при создании или изменении общего ресурса:
```
net share SHARE=D:\Share /users:5
```
или изменить существующую шару:
```
net share SHARE /users:5
```
👉 Это означает: максимум **5 одновременных подключений**
## 2. Через GUI (если Windows Server)
Если у тебя Windows Server:
1. ПКМ на папке → **Свойства**
2. Вкладка **Общий доступ**
3. **Расширенная настройка**
4. Поставь:
    - ✔ “Ограничить число пользователей”
    - Укажи число (например 5)
---
## 🧠 3. Через PowerShell (Server-ориентировано)
```
Set-SmbShare -Name "SHARE" -ConcurrentUserLimit 5
```
👉 Требует Windows Server или Windows Pro с SMB Server компонентами
---
## 🔎 4. Проверка текущих подключений
```
Get-SmbSession
```
или
```
net session
```
---
## 🚨 Важные нюансы
- Ограничение работает на **подключение к шару**, а не на файлы внутри неё
- Если пользователь уже подключился — он занимает слот до разрыва сессии
- При достижении лимита новые пользователи получат ошибку:
    > “No more connections can be made to this remote computer”
